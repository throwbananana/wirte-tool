# Writer Tool 实施计划

## 状态更新

本文最初用于记录功能补全方向。当前仓库已经进入“收口优先”的阶段，真实优先级应以 `README.md`、`docs/testing/RELEASE_BLOCKERS.md`、`docs/testing/BETA_VALIDATION.md` 和现有测试主干为准。

本轮调整后的判断如下：

- 主应用、工具入口、核心导出链路和基础测试体系已经落地
- 当前最重要的工作是仓库卫生、文档同步、安装路径说明、CI 和发布流程
- 仍在推进的功能不再默认视为“即将上线”，而是明确区分为 Beta 验证项

## 已落地能力

- 多入口启动：主应用、工具中心、资产编辑器、事件编辑器、事件分析器
- 主体模块分层：`writer_app/core`、`writer_app/controllers`、`writer_app/ui`
- 导出相关能力与测试覆盖已经存在，并纳入主线文档
- 控制器、命令模式、历史记录和项目服务已经形成基础维护结构
- 面向发布的文档、CI 和清单已开始收口

## 当前收口重点

1. 清理仓库根目录，移除缓存、临时文件和重复占位文档
2. 用 README 和测试文档明确 Stable / Beta 边界
3. 以 `python -m unittest discover tests` 作为最小自动化质量信号
4. 为安装、打包和发版准备统一清单

## 仍在推进

- 关系图、项目对话、浮动助手等高级工作流的 Beta 验证
- Windows 打包与对外下载物整理
- 公共截图、GIF 和更完整的发布说明
- 可选音频/语音依赖的安装体验优化

## 历史计划视角

以下方向仍然有价值，但它们现在属于“后续增量”而不是当前发版闸门：

### 导出增强

- 更细粒度的数据表导出
- 分角色台词本与角色定向导出

### 沉浸与生产力

- 白噪音 / 环境音播放器
- 写作冲刺与更多游戏化反馈

### 深度关联分析

- 关系线到场景的反向查询
- 人物共现和关联跳转强化

如果继续扩展这些方向，应优先补对应测试和用户可见说明，再考虑扩大默认支持范围。
